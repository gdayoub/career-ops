## F) STAR+R Interview Stories

**1. Tell me about an applied ML project you built and deployed**
- **S:** Needed to classify Arabic geopolitical news by topic with no labeled dataset
- **T:** Build and deploy a working classifier from raw text to production
- **A:** Labeled a training dataset manually from known examples; extracted TF-IDF features after Arabic preprocessing; trained logistic regression with cross-validation; integrated into production pipeline
- **R:** Live classifier running continuously, deployed with Streamlit dashboard
- **R:** Would use a pre-trained multilingual model for better accuracy on novel categories

**2. Tell me about working with complex, messy data (C3 AI core challenge)**
- **S:** Seismos PJR PDFs had inconsistent structure — field names, units, and layouts varied by engineer
- **T:** Extract clean structured data from unstructured documents reliably
- **A:** Built layered Python extractors (text → positional → regex heuristic); validated each field type against known data; flagged anomalies for human review
- **R:** Fully automated extraction used in client-facing QC; zero data integrity failures during my tenure
- **R:** Would train a document layout classifier to handle novel PDF formats automatically

**3. Tell me about a time you worked independently on a defined goal**
- **S:** At Seismos, the full PDF extraction + SQL reporting pipeline was scoped to me alone
- **T:** Deliver a working system in 10 weeks with no prior documentation
- **A:** Broke the project into: understand data → extract fields → validate → store → report; set weekly milestones; shipped to engineering team in week 4
- **R:** Pipeline delivered ahead of schedule; SQL reporting layer added in remaining weeks
- **R:** Setting explicit weekly milestones was the key to staying on track solo

**4. Tell me about working at the intersection of ML and business problems**
- **S:** CoverMe's AI generation needed to optimize for ATS (machine criteria) and recruiter quality (human criteria) simultaneously
- **T:** Build a prompt system that satisfied both
- **A:** Decomposed the problem: keywords extraction (ATS), sentence complexity targeting (readability), tone alignment (recruiter preference); baked each into the prompt context
- **R:** Users report higher interview callback rates; letters pass ATS filters more reliably
- **R:** ML/AI for business problems requires defining the right evaluation criteria first

**5. Why C3 AI? (Motivation)**
- The intersection of enterprise-scale data and ML is where the hardest unsolved problems live — messy data, diverse formats, customer-specific constraints, and real consequences (predictive maintenance = machines don't break; fraud detection = people don't lose money)
- C3 AI applies exactly the techniques I've built (classification, regression, production ML pipelines) at a scale I couldn't access otherwise as a junior
- The Ascend Program's structure — working directly with customers — teaches the "last mile" of DS that's missing from most internships
